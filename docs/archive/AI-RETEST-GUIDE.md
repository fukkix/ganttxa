# AI 解析功能重新测试指南

## 问题分析

### 原始问题
AI 解析 Excel 文件后，任务成功导入但缺少日期信息。

### 根本原因
1. Excel 中的日期格式为 `4/15/26`（月/日/年的两位数年份）
2. AI prompt 对这种简写格式的识别不够明确
3. 后端默认值处理会自动填充今天的日期，掩盖了解析失败的问题

#### 添加 Claude Sonnet 4.6
- 最新的 Claude 模型，智能程度显著提升
- 对复杂日期格式的识别能力更强
- 更好的表格结构理解能力
- 设为默认推荐模型

### 1. 增强 AI Prompt (aiParser.ts)

#### 日期格式识别
添加了详细的日期格式说明：
- `4/15/26` → `2026-04-15`（两位数年份处理）
- `M/D/YY` → 无前导零格式
- Excel 序列号格式
- `-` 或空值 → `null`

#### 解析示例
添加了完整的输入输出示例，帮助 AI 理解表格结构：
```
输入: 4/15/26
输出: "2026-04-15"
```

#### 特别注意事项
- 明确两位数年份的处理规则
- 结束日期为 `-` 时设为 null
- 多个单位时取第一个作为负责人

### 2. 改进默认值处理

#### 之前的逻辑
```typescript
startDate: task.startDate || new Date().toISOString().split('T')[0]
```
问题：自动填充今天的日期，用户无法知道日期未被正确解析

#### 现在的逻辑
```typescript
startDate: task.startDate || ''
```
优势：保持为空，触发前端的警告提示

### 3. 增强调试日志

添加了详细的任务字段日志：
```typescript

  name: task.name,
  startDate: task.startDate,
  endDate: task.endDate,

```

**解决方案**：

## 后续优化建议

### 1. 添加日期格式后处理
在后端添加日期格式验证和转换：
```typescript
function normalizeDate(dateStr: string): string | null {
  if (!dateStr || dateStr === '-') return null
  
  // 处理 M/D/YY 格式
  const match = dateStr.match(/^(\d{1,2})\/(\d{1,2})\/(\d{2})$/)
  if (match) {
    const [, month, day, year] = match
    const fullYear = `20${year}`
    return `${fullYear}-${month.padStart(2, '0')}-${day.padStart(2, '0')}`
  }
  
  return dateStr
}
```

### 2. 添加日期验证
```typescript
function isValidDate(dateStr: string): boolean {
  const date = new Date(dateStr)
  return date instanceof Date && !isNaN(date.getTime())
}
```

### 3. 提供日期修正建议
当检测到日期格式问题时，提供修正建议给用户。
