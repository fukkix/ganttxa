# 无日期任务处理功能

## 问题描述
AI 解析的任务成功导入到项目中，但部分任务缺少 startDate，导致甘特图无法正常显示这些任务。

## 解决方案

### 1. 甘特图组件 (GanttChart.tsx)

#### 任务分类
- 将任务分为有日期和无日期两类
- 只使用有日期的任务计算时间范围

#### 全部无日期处理
如果所有任务都没有日期：
- 显示居中的警告提示
- 提示用户需要设置日期
- 显示缺少日期的任务数量

#### 部分无日期处理
如果部分任务没有日期：
- 在表头下方显示黄色警告条
- 显示缺少日期的任务数量
- 在时间轴右侧为无日期任务绘制特殊标记：
  - 黄色虚线框
  - 黄色背景
  - "⚠️ 未设置日期" 文字提示

### 2. 任务列表组件 (TaskList.tsx)

#### 视觉标记
- 在任务名称旁边添加 "⚠️ 缺少日期" 标签（黄色）
- 在日期显示区域显示 "⚠️ 未设置日期" 提示

#### 样式
- 使用警告色系（黄色）突出显示
- 与里程碑标签样式保持一致

### 3. 样式配置 (tailwind.config.js)

添加警告色系：
- `warning`: #F59E0B
- `warning-container`: #FEF3C7
- `on-warning`: #ffffff
- `on-warning-container`: #92400E

## 用户体验

### 视觉反馈
1. 顶部警告条：快速了解有多少任务缺少日期
2. 甘特图标记：在时间轴右侧清晰标记无日期任务
3. 任务列表标签：在侧边栏中直接看到哪些任务需要设置日期

### 操作引导
- 提示用户在任务列表中编辑任务
- 引导用户添加开始日期
- 保持界面友好，不阻断正常使用

## 技术细节

### Canvas 绘制
- 使用虚线框 `setLineDash([4, 4])` 绘制警告标记
- 使用 `roundRect` 绘制圆角矩形
- 警告标记固定在右侧，不受时间轴滚动影响

### 布局调整
- 警告条高度：35px
- 调整后续元素的 Y 坐标偏移
- 今日线起始位置考虑警告条高度

### 颜色一致性
- 统一使用 Material Design 警告色
- 与错误色（红色）区分
- 保持品牌色系协调
